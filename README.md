School Management System - (MIS Department Project)

وصف المشروع (Project Description)
نظام إدارة مدرسة متكامل يعتمد على تقنيات الويب (PHP & MySQL)، يهدف إلى أتمتة العمليات الإدارية المتعلقة بالطلاب، المعلمين، والمواد الدراسية. يوفر النظام لوحة تحكم للمسؤول لإدارة قاعدة البيانات وضمان سير العمل الأكاديمي بكفاءة.

فكرة المشروع (Project Concept)
فكرة المشروع تقوم على حل المشاكل الإدارية التقليدية في المدارس من خلال التحول الرقمي. يتيح النظام إضافة، عرض، تعديل، وحذف بيانات الطلاب والمعلمين (CRUD Operations)، مما يقلل من الأخطاء البشرية ويسهل الوصول إلى البيانات بشكل فوري وآمن.

المتطلبات التقنية (Tech Stack)
• Backend: PHP
• Database: MySQL (phpMyAdmin)
• Frontend: HTML/CSS (Bootstrap)
• Server: XAMPP

الجداول المستخدمة (Database Structure)
يتكون النظام من 5 جداول رئيسية لضمان تنظيم البيانات:
1. users: لتخزين بيانات المستخدمين وصلاحياتهم.
2. students: لتخزين بيانات الطلاب.
3. teachers: لتخزين بيانات المعلمين.
4. subjects: لتخزين المواد الدراسية.
5. classes: لتخزين بيانات الفصول.

هيكلية قاعدة البيانات (Database Schema)

يتكون النظام من 5 جداول رئيسية، تم تصميمها لضمان تكامل البيانات وتنظيم العلاقات بينها باستخدام المفاتيح الأساسية والخارجية.

1. جدول المستخدمين (Users)

يحتوي على بيانات المسؤولين والمستخدمين، ويتضمن الحقول التالية:

- ID
- username
- password
- role

يستخدم هذا الجدول لإدارة حسابات المستخدمين وصلاحيات الدخول إلى النظام.

2. جدول الطلاب (Students)

يحتوي على البيانات الأساسية للطلاب، ويتضمن الحقول التالية:

- ID
- name
- email
- class_id

يمثل الحقل class_id مفتاحًا خارجيًا (Foreign Key) يربط كل طالب بالفصل الدراسي الخاص به.

3. جدول المعلمين (Teachers)

يحتوي على بيانات المعلمين، ويتضمن الحقول التالية:

- ID
- name
- specialization

يستخدم لإدارة بيانات الكادر الأكاديمي.

4. جدول المواد الدراسية (Subjects)

يحتوي على بيانات المواد الدراسية، ويتضمن الحقول التالية:

- ID
- subject_name
- teacher_id

يمثل الحقل teacher_id مفتاحًا خارجيًا (Foreign Key) يربط المادة بالمعلم المسؤول عن تدريسها.

5. جدول الفصول (Classes)

يحتوي على بيانات الفصول الدراسية، ويتضمن الحقول التالية:

- ID
- class_name

يستخدم لتنظيم الفصول الدراسية وتوزيع الطلاب عليها.

ملاحظة

تم تصميم العلاقات (Relationships) بين الجداول باستخدام المفاتيح الأساسية (Primary Keys) والمفاتيح الخارجية (Foreign Keys)؛ وذلك لضمان سلامة البيانات، ومنع التكرار، وتحقيق التكامل بين جميع الكيانات (Entities) داخل النظام.

شرح شاشات النظام (System Screens)

يتضمن النظام مجموعة من الشاشات الرئيسية التي تتيح للمستخدم إدارة البيانات بسهولة وكفاءة، وهي كما يلي:

1. شاشة تسجيل الدخول (Login Screen)

تعد البوابة الرئيسية للنظام، حيث يتم من خلالها التحقق من صحة بيانات المستخدم (Validation)، واستخدام الجلسات (Sessions) لضمان تسجيل الدخول الآمن وإدارة صلاحيات المستخدم.

2. لوحة تحكم المسؤول (Admin Dashboard)

تمثل المركز الرئيسي لإدارة النظام، وتوفر للمسؤول إمكانية الوصول إلى جميع أقسام النظام وإدارة البيانات المختلفة من مكان واحد.

3. شاشات إدارة البيانات (CRUD Screens)

تتضمن شاشات مخصصة لإدارة بيانات النظام، وتتيح للمسؤول تنفيذ العمليات الأساسية التالية:

- إضافة (ADD): إدخال بيانات جديدة إلى قاعدة البيانات.
- عرض (Read): استعراض البيانات والسجلات المخزنة داخل النظام.
- حذف (Delete): إزالة السجلات التي لم تعد مطلوبة من قاعدة البيانات.





طريقة التشغيل (How to Run)
1. تأكد من تثبيت وتشغيل XAMPP (Apache & MySQL).
انشأ ملف داخل htdocs وقم بتسميته school-management
2. قم بنقل مجلد المشروع إلى المسار: C:\xampp\htdocs\school-management
3. افتح phpMyAdmin وأنشئ قاعدة بيانات باسم (school_db).
4. قم باستيراد ملف قاعدة البياناتC:\xampp\htdocs\school-management\database.sql\database.sql الموجود داخل مجلد المشروع.
5. افتح المتصفح وتوجه إلى الرابط: http://localhost/school-management/login.php
